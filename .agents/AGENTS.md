## 하네스: Network Storage

**목표:** Go 백엔드(`backend/`)와 Fyne 프론트엔드(`fyne-frontend/`)의 효율적인 연동 개발 및 변경사항의 자동화된 검증

**트리거:** Network Storage 프로젝트의 코드 구현, 버그 수정, 구조 변경 등 실무 작업 요청 시 `network-storage-orchestrator` 스킬을 사용하라. 단순 질문은 직접 응답 가능.

**절대 규칙:** 메인 에이전트(오케스트레이터)는 절대로 단 한 줄의 코드도 직접 수정( replace_file_content  등)하지 마라. 아무리 사소한 버그 수정이거나 단순 질문에서 파생된 이슈일지라도, 코드를 변경해야 한다면 반드시  network-storage-orchestrator  워크플로우를 가동하여 하위 에이전트(Frontend/Backend Developer)를 스폰(invoke)한 뒤 수정을 위임하라.
